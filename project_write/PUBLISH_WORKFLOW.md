# 文档发布流程约定（猫老板）

更新时间：2026-03-26

## 固定流程（后续默认执行）
1. 本地先产出 Markdown 文档，保存到 `project_write/` 目录。
2. 同步推送到 GitHub 仓库（仓库已打通）。
3. 同步推送到 Notion：
   - 写入位置：`势通工作文档中心` 数据表内，作为与「工作汇总」同级的新条目。
   - 不再写到「工作汇总」条目内。
   - 正文格式：使用 Notion 原生 block（标题/段落/列表/代码块/引用等），不使用内嵌网页方式。
4. 中文写入注意事项：
   - 采用 UTF-8/Unicode 安全写入方式，避免 PowerShell 编码导致乱码。

## 默认元信息（可按需调整）
- 文档名称：与 Markdown 标题一致
- 类别：默认 `记录文档`（如未指定）

## 执行口令（你一句话我就开工）
- “按发布流程发这篇文档”
- “按默认流程生成+推 GitHub+同步 Notion”
